## Moduł: CameraView (Expo SDK 54)
- `<CameraView>` w SDK 54 NIE obsługuje children. Cały overlay UI musi być rodzeństwem `<CameraView>`, nie dzieckiem.
- Prop `zoom` przyjmuje wartości 0-1 gdzie 0 = standardowy obiektyw (1x). Expo Camera nie daje dostępu do przełączania fizycznych obiektywów (szerokokątny).
- Przycisk `0.5x` (szerokokątny) jest mylący i został usunięty. Etykiety: 1x (zoom=0), 2x (zoom=0.03), 3x (zoom=0.06).

## Moduł: Zapis Plików
- `expo-media-library` NIE MOŻE być importowane (nawet nieużywane) — sam import ładuje moduł natywny który wymaga uprawnień AUDIO w AndroidManifest, powodując crash.
- Zapis odbywa się wyłącznie przez `expo-sharing` (Sharing.shareAsync).

## Moduł: Przetwarzanie Obrazu (ROZWIĄZANO)
- `imageProcessor.ts` → `parsePNGBytes()` teraz poprawnie dekompresuje PNG IDAT chunks używając wbudowanego dekodera zlib/inflate (`src/algorithms/inflate.ts`).
- Dekoder inflate napisany w czystym TypeScript (RFC 1950/1951), zero zewnętrznych zależności.
- Obsługuje: stored blocks, fixed Huffman, dynamic Huffman, PNG scanline unfiltering (None, Sub, Up, Average, Paeth).
- Obsługiwane formaty: 8-bit RGB (colorType=2) i 8-bit RGBA (colorType=6). Inne spadają do syntetycznych danych z logiem ostrzeżenia.
- Pomiary Canny Edge i ArUco operują teraz na PRAWDZIWYCH danych pikselowych ze zdjęcia.
